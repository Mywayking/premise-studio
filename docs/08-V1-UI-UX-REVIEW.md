## 二、P0 问题（影响核心功能可用性）

### 2.1 AI 返回的结构化 JSON 直接当成纯文本展示

**严重程度：🔴 阻断**

AI 各阶段返回的都是结构化 JSON，但前端完全不解析：

**前提阶段** — AI 返回：
```json
{"premises":[{"statement":"人们总说要在工作中找到热爱，但更多人找到的只是对周末的热爱。","attitude":"对职场鸡汤的讽刺","conflict":"理想主义的职业建议 vs 现实中的摸鱼心态","potential":"可以展开上班摸鱼、下班狂欢的对比"}]}
```

用户看到的是：编辑器里一坨 raw JSON。

**角度阶段** — AI 返回：
```json
{"angles":[{"type":"字面化","description":"...","opening":"...","direction":"..."}]}
```

同样一坨 raw JSON。

**创作树侧栏** — 卡片标题是 `card.content.slice(0, 40)`，显示为：
```
前提 {"premises":[ {"statement":"人们总说要在工作中找
```
完全无法用于导航。

**期望行为：**
- 前提阶段 → AI 返回 3-5 个 premise → 自动拆成 5 张独立的 premise 卡片（每个子节点）
- 角度阶段 → AI 返回 3-5 个 angle → 自动拆成 5 张独立的 angle 卡片
- 卡片展示应解析 JSON，展示 statement/attitude/conflict/potential 等字段
- 卡片标题使用 statement 的前 30 字，而不是 raw JSON

### 2.2 行标注功能桌面端体验差、移动端完全不可用

**严重程度：🔴 阻断（移动端）**

行标注的 select 选择器使用 `opacity-0 group-hover:opacity-100`，只有在 hover 时才显示。移动端没有 hover，所以这个功能在手机上完全无法使用。

同时 select 字体为 `text-[10px]`，包含 8 个选项（开场/铺垫/炸点/Tag/Topper/表演/过渡/收尾），在桌面端也过小难以操作。

**期望行为：**
- 移动端：始终显示选择器（不用 hover）
- 桌面端：增大字体到至少 12px，或使用更直观的交互方式（如点击行号弹出 popover）

## 三、P1 问题（影响用户体验）

### 3.1 Session 名字全是"新会话"，无法区分

首页 Session 列表里所有 session 都叫"新会话"，区分不了哪个是哪个。

**期望行为：**
- 支持双击标题或点击编辑图标来重命名
- 自动从 session 根素材中提取前 20 字作为默认名称

### 3.2 没有删除/重命名卡片和 Session

整个界面没有任何删除按钮、右键菜单、或重命名入口。用户误操作后无法清理。

**期望行为：**
- Session 列表项支持右键或更多按钮 → 删除/重命名
- Card Tree 节点支持右键或三点菜单 → 删除/重命名/复制
- 删除时有确认弹窗（card 下可能有子节点）

### 3.3 没有 AI 请求的即时视觉反馈

点击"提炼前提"后，按钮没有任何即时变化。右侧面板底部的 "AI 正在生成中..." 要等流式数据到达后才出现，中间有几秒空白期用户不知道发生了什么。

**期望行为：**
- 点击后按钮立即变为 disabled + loading 状态
- 或者编辑器区域立即显示流式 token，而不是等数据到达

### 3.4 "保存"按钮语义不清

编辑器上方有"保存"按钮，但内容改动已通过 Zustand 自动同步到 localStorage。用户会困惑：我需要点保存吗？什么时候该点？

**期望行为：**
- 自动保存 + "已自动保存" 时间戳提示
- 或去掉按钮，改用手动触发（Cmd+S）并给出 toast

### 3.5 首页空状态缺少引导

空状态只有一句话："还没有创作记录，在上方输入灵感开始第一个 session"。
没有产品介绍、创作流程说明、示例素材、模板等。

**期望行为：**
- 展示 3 步创作流程（捕获 → AI 提炼 → 打磨）
- 提供一个"试试示例"按钮，点击后加载预置素材走一遍完整流程
- 或至少展示一张示意图

## 四、P2 问题（影响产品完整性）

### 4.1 首页全客户端渲染，SEO 为零

`<title>Premise Studio | 单口喜剧创作 IDE</title>` 存在，但页面内容全在客户端渲染。搜索引擎只能看到 loading spinner。

**期望行为：**
- 首页至少 SSR 渲染标题和描述
- 或添加 `<meta name="description">` 和 og 标签

### 4.2 没有 Undo/Redo

AI 生成的内容替换了现有内容后无法回退。Zustand store 没有历史栈。

**期望行为：**
- 编辑器支持 Cmd+Z / Cmd+Shift+Z
- 或每个 card 保留版本历史

### 4.3 没有导出功能

创作完成后无法导出为文本、Markdown、JSON 或分享链接。

**期望行为：**
- 导出整个 session 为 Markdown
- 导出单个 draft 为纯文本（去除标注）
- 可选：生成分享链接

### 4.4 没有键盘快捷键

纯鼠标操作，缺少常用快捷键：
- `Cmd+S` 保存
- `Cmd+Z` 撤销
- `Cmd+Enter` 触发 AI 动作
- `Cmd+B` 切换左侧面板
- `Cmd+Shift+B` 切换右侧面板

### 4.5 移动端布局需优化

当前移动端三栏同时可见（虽然用了 drawer 系统），但关闭一个面板后没有明显提示告诉用户如何重新打开。测试中移动端树和 AI 面板同时渲染，占用大量空间。

**期望行为：**
- 移动端默认隐藏左右面板
- 底部 tab bar 切换（树/编辑器/陪练）
- 或顶部固定按钮始终可见

### 4.6 卡片内容无格式区分

所有卡片类型的编辑区都是纯文本框。不同类型的卡片需要不同的展示方式：

| 卡片类型 | 当前 | 期望 |
|----------|------|------|
| material | textarea | textarea（OK） |
| premise | 显示 raw JSON | 卡片展示 statement/attitude/conflict，可展开 |
| angle | 显示 raw JSON | 卡片展示 type/opening/direction，可选择继续 |
| draft | textarea | textarea + 行标注（OK） |
| rewrite | textarea | textarea + diff 对比原版 |

### 4.7 缺少暗色模式

全白界面，长时间盯着编辑段子对眼睛不友好。

## 五、V2 优先修复建议

### 第一步：核心体验修复（必须）

1. **解析 AI 返回的 JSON** — 把 premise/angle 的 JSON 拆成独立卡片，这是整个产品质量断崖式提升的关键
2. **修复行标注的移动端可用性** — 去掉 hover 依赖
3. **添加 AI 请求的即时反馈** — 点击后立即显示 loading

### 第二步：基础交互完善

4. Session 重命名和删除
5. Card 删除和重命名
6. 保存按钮改为自动保存 + 时间戳提示
7. 空状态引导

### 第三步：产品化

8. 首页 SSR / SEO
9. Undo/Redo
10. 导出功能
11. 键盘快捷键
12. 暗色模式
